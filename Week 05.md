# Week 05 - Dynamic Analysis

## 📚 Topik
- Dynamic Analysis
- Perilaku Program Saat Runtime
- Perbedaan Static dan Dynamic Analysis

## 📝 Ringkasan Pembelajaran

Pada minggu ini saya mempelajari Dynamic Analysis sebagai metode untuk mengamati perilaku suatu program ketika sedang dijalankan. Berbeda dengan Static Analysis yang berfokus pada struktur program, Dynamic Analysis memungkinkan analis melihat aktivitas program secara langsung, seperti interaksi dengan sistem operasi, penggunaan memori, akses file, hingga komunikasi jaringan. Saya juga mempelajari perbedaan karakteristik antara analisis statis dan dinamis serta bagaimana kedua metode tersebut saling melengkapi dalam proses Reverse Engineering.

## ✅ Apa yang Dipahami

Saya memahami bahwa Dynamic Analysis sangat berguna untuk mengetahui perilaku aktual suatu program ketika dijalankan. Saya juga memahami bahwa debugger merupakan salah satu tools utama dalam Dynamic Analysis karena memungkinkan analis mengamati jalannya instruksi program secara bertahap dan menemukan proses penting yang tidak dapat diketahui melalui analisis statis saja.

## ❓ Apa yang Masih Membingungkan

Saya masih ingin mempelajari lebih lanjut mengenai teknik debugging tingkat lanjut seperti breakpoint bersyarat, API hooking, dan memory dumping. Selain itu, saya juga ingin memahami bagaimana malware dapat mendeteksi keberadaan debugger dan menghindari proses analisis.
