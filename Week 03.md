# Week 03 - Binary dan Executable

## 📚 Topik
- Binary
- Executable
- Tahapan Kompilasi
- Format File Executable

## 📝 Ringkasan Pembelajaran

Pada minggu ini saya mempelajari konsep dasar binary dan executable sebagai fondasi dalam Reverse Engineering. Binary merupakan kumpulan instruksi dalam bentuk *machine code* yang dapat dipahami langsung oleh prosesor, sedangkan executable adalah file hasil proses kompilasi yang memiliki struktur tertentu sehingga dapat dijalankan oleh sistem operasi. Saya juga mempelajari tahapan pembentukan executable mulai dari preprocessing, compiling, assembling, hingga linking sehingga dapat memahami bagaimana sebuah program dikembangkan sebelum akhirnya menjadi file yang dapat dieksekusi.

## ✅ Apa yang Dipahami

Saya memahami bahwa executable memiliki struktur yang berbeda pada setiap sistem operasi, seperti PE pada Windows, ELF pada Linux, dan Mach-O pada macOS. Saya juga memahami bahwa proses kompilasi terdiri dari beberapa tahapan yang saling berkaitan hingga menghasilkan file executable. Pengetahuan ini menjadi dasar sebelum mempelajari proses analisis program menggunakan teknik Reverse Engineering.

## ❓ Apa yang Masih Membingungkan

Saya masih ingin memahami secara lebih rinci bagaimana proses linking bekerja dan bagaimana sistem operasi memuat executable ke dalam memori ketika program dijalankan. Saya juga ingin mempelajari lebih lanjut mengenai proses loading yang dilakukan oleh operating system.
